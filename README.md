# 🌞 GridMind AI Platform

> ระบบบริหารจัดการพลังงานอัจฉริยะและการคาดการณ์การผลิตไฟฟ้าจากโซลาร์เซลล์ด้วยเทคโนโลยีปัญญาประดิษฐ์

GridMind AI Platform เป็นโปรเจกต์ที่พัฒนาขึ้นเพื่อแก้ไขปัญหาข้อมูลที่กระจัดกระจายของ **บริษัท ลีโอนิกส์ จำกัด** โดยรวบรวม วิเคราะห์ และพยากรณ์ข้อมูลด้านพลังงาน (การผลิตไฟฟ้าจากโซลาร์เซลล์ และการใช้ไฟฟ้า) ด้วยสถาปัตยกรรมแบบ **Edge–Cloud** ร่วมกับโมเดล Machine Learning **LightGBM** โดยอ้างอิงข้อมูลสภาพอากาศจาก **Open-Meteo API**

จุดเด่นของระบบคือกระบวนการ **Automated Retraining** ที่ปรับปรุงความแม่นยำของโมเดลโดยอัตโนมัติทุกวัน ผ่านการวัดค่าความคลาดเคลื่อนด้วย **MAPE** และ **RMSE** พร้อมเว็บแดชบอร์ดที่พัฒนาด้วย **React** สำหรับแสดงผลข้อมูลเชิงลึก รายงานการลดก๊าซเรือนกระจก และการแจ้งเตือนความผิดปกติผ่าน **LINE**

## 🔍 ภาพรวมโปรเจกต์

บริษัท ลีโอนิกส์ จำกัด ประสบปัญหาข้อมูลด้านพลังงานที่กระจัดกระจายอยู่หลายแหล่ง ทำให้ยากต่อการนำมาใช้วิเคราะห์และวางแผนการผลิต/ใช้พลังงานอย่างมีประสิทธิภาพ โปรเจกต์นี้จึงถูกพัฒนาขึ้นเพื่อ:

- รวบรวมข้อมูลการผลิตไฟฟ้าจากโซลาร์เซลล์และข้อมูลสภาพอากาศไว้ในที่เดียว
- ใช้โมเดล Machine Learning พยากรณ์ปริมาณการผลิตไฟฟ้าและการใช้ไฟฟ้าล่วงหน้า
- ปรับปรุงความแม่นยำของโมเดลอย่างต่อเนื่องโดยไม่ต้องมีคนเข้ามาแทรกแซง (Automated Retraining)
- นำเสนอข้อมูลผ่านแดชบอร์ดที่เข้าใจง่าย พร้อมแจ้งเตือนแบบเรียลไทม์

## ✨ ฟีเจอร์หลัก

- **📊 พยากรณ์การผลิตไฟฟ้าโซลาร์เซลล์** — ใช้โมเดล LightGBM ร่วมกับข้อมูลสภาพอากาศเพื่อพยากรณ์ปริมาณไฟฟ้าที่จะผลิตได้
- **⚡ พยากรณ์การใช้ไฟฟ้า** — วิเคราะห์แนวโน้มการใช้พลังงานเพื่อวางแผนล่วงหน้า
- **🌤️ ดึงข้อมูลสภาพอากาศแบบเรียลไทม์** — เชื่อมต่อกับ Open-Meteo API
- **🔁 Automated Retraining** — เทรนโมเดลใหม่อัตโนมัติทุกวัน พร้อมวัดผลด้วยค่า MAPE และ RMSE เพื่อรักษาความแม่นยำของการพยากรณ์
- **🖥️ เว็บแดชบอร์ด (React)** — แสดงข้อมูลเชิงลึก กราฟแนวโน้ม และสรุปผลการพยากรณ์
- **🌱 รายงานการลดก๊าซเรือนกระจก** — คำนวณและแสดงผลกระทบเชิงสิ่งแวดล้อมจากการใช้พลังงานแสงอาทิตย์
- **🔔 ระบบแจ้งเตือนผ่าน LINE** — แจ้งเตือนความผิดปกติของระบบหรือค่าพยากรณ์ที่เบี่ยงเบนไปจากปกติ
- **📄 Export รายงาน** — สร้างรายงาน PDF จากแดชบอร์ด (รองรับด้วย `jspdf` และ `html2canvas`)

## 🏗️ สถาปัตยกรรมระบบ

ระบบใช้สถาปัตยกรรมแบบ **Edge–Cloud**:

```
[ อุปกรณ์/เซนเซอร์หน้างาน ]
          │  ส่งข้อมูลดิบ (Edge)
          ▼
[ Backend Server (Express + MySQL) ]
          │
          ├──► [ Open-Meteo API ]  → ข้อมูลสภาพอากาศ
          │
          ├──► [ LightGBM Forecast Model ] → พยากรณ์การผลิต/ใช้ไฟฟ้า
          │         │
          │         └──► Automated Retraining (รายวัน, วัดด้วย MAPE / RMSE)
          │
          ├──► [ LINE Notify / Messaging API ] → แจ้งเตือนความผิดปกติ
          │
          ▼
[ React Dashboard ] → แสดงผลข้อมูลเชิงลึก, กราฟ, รายงาน CO₂, Export PDF
```

## 🚀 การติดตั้งและเริ่มต้นใช้งาน

### สิ่งที่ต้องมีก่อน (Prerequisites)

- [Node.js](https://nodejs.org/) (แนะนำเวอร์ชัน 18+ เนื่องจากใช้ React 19 / Vite 7)
- [MySQL](https://www.mysql.com/) (สำหรับฐานข้อมูลฝั่ง backend)
- [Python 3](https://www.python.org/) พร้อม `lightgbm` และไลบรารีที่เกี่ยวข้อง (สำหรับรันส่วนโมเดลพยากรณ์ในโฟลเดอร์ `Test forecast model/`)

### ขั้นตอนการติดตั้ง

1. **Clone โปรเจกต์**
   ```bash
   git clone https://github.com/PleumAndNight/SProject.git
   cd SProject
   ```

2. **ติดตั้ง dependencies ฝั่ง Frontend/Backend (Node.js)**
   ```bash
   npm install
   ```

3. **ตั้งค่าฐานข้อมูล MySQL**
   - สร้างฐานข้อมูลใหม่
   - ตั้งค่าการเชื่อมต่อผ่าน Environment Variables (ดูหัวข้อด้านล่าง)

4. **ตั้งค่าไลบรารี Python สำหรับโมเดลพยากรณ์** (ถ้าต้องการรัน/พัฒนาโมเดลต่อ)
   ```bash
   cd "Test forecast model"
   pip install lightgbm pandas numpy scikit-learn requests
   ```

5. **รัน Backend Server**
   ```bash
   node server.js
   ```

6. **รัน Frontend (Development mode)**
   ```bash
   npm run dev
   ```

7. เปิดเบราว์เซอร์ไปที่ `http://localhost:5173` (พอร์ตเริ่มต้นของ Vite)

### Scripts ที่ใช้ได้ (จาก `package.json`)

| คำสั่ง | คำอธิบาย |
|---|---|
| `npm run dev` | รัน Vite dev server สำหรับพัฒนา frontend |
| `npm run build` | Build โปรเจกต์สำหรับ production |
| `npm run preview` | ดูตัวอย่างผลลัพธ์หลัง build |
| `npm run lint` | ตรวจสอบคุณภาพโค้ดด้วย ESLint |

## 🔐 การตั้งค่า Environment Variables

โปรเจกต์นี้ต้องเชื่อมต่อกับฐานข้อมูล MySQL และ LINE API จึงควรมีไฟล์ `.env` (ไม่ commit เข้า repo) เก็บค่าต่าง ๆ เช่น:

```env
# Database
DB_HOST=localhost
DB_USER=your_user
DB_PASSWORD=your_password
DB_NAME=gridmind_db

# Server
PORT=5000

# LINE Notify / Messaging API
LINE_CHANNEL_ACCESS_TOKEN=your_line_token

# Open-Meteo API (โดยทั่วไปไม่ต้องใช้ API key แต่ระบุ endpoint ที่ใช้ไว้เพื่อความชัดเจน)
OPEN_METEO_API_URL=https://api.open-meteo.com/v1/forecast
```


## 📖 วิธีใช้งาน

1. เข้าสู่ระบบ/ลงทะเบียนผู้ใช้งาน (รองรับการเข้ารหัสรหัสผ่านด้วย bcryptjs)
2. ระบบจะดึงข้อมูลสภาพอากาศจาก Open-Meteo API โดยอัตโนมัติ
3. โมเดล LightGBM จะประมวลผลและพยากรณ์การผลิตไฟฟ้าโซลาร์เซลล์ / การใช้ไฟฟ้า
4. ผลการพยากรณ์จะแสดงผลผ่านแดชบอร์ด React พร้อมกราฟและรายงาน CO₂
5. หากระบบตรวจพบความผิดปกติ (เช่น ค่าพยากรณ์เบี่ยงเบนมาก) จะส่งการแจ้งเตือนผ่าน LINE
6. ทุกวัน ระบบจะทำการ Retrain โมเดลใหม่โดยอัตโนมัติ และประเมินผลด้วย MAPE/RMSE เพื่อรักษาความแม่นยำ
7. สามารถ Export รายงานสรุปเป็นไฟล์ PDF ได้จากหน้าแดชบอร์ด

## 📊 ชุดข้อมูล (Dataset)

ไฟล์ `cleaned_solar_dataset_multi_months.csv` เป็นชุดข้อมูลโซลาร์เซลล์ที่ผ่านการทำความสะอาดแล้ว ครอบคลุมหลายเดือน ใช้สำหรับ:

- เทรนโมเดล LightGBM เบื้องต้น
- ทดสอบความแม่นยำของโมเดลก่อนนำไปใช้งานจริง
- เป็นข้อมูลอ้างอิงสำหรับกระบวนการ Automated Retraining

## 🤖 โมเดลพยากรณ์

โฟลเดอร์ `Test forecast model/` เก็บโค้ดที่ใช้พัฒนาและทดสอบโมเดลพยากรณ์ โดยมีแนวทางหลักดังนี้:

- **อัลกอริทึม:** LightGBM (Gradient Boosting)
- **Input:** ข้อมูลสภาพอากาศจาก Open-Meteo API + ข้อมูลย้อนหลังจากชุดข้อมูลโซลาร์เซลล์
- **Output:** ค่าพยากรณ์การผลิตไฟฟ้าโซลาร์เซลล์ และการใช้ไฟฟ้า
- **การวัดผล:** MAPE (Mean Absolute Percentage Error) และ RMSE (Root Mean Squared Error)
- **กระบวนการ Retraining:** รันอัตโนมัติเป็นรายวัน เพื่อให้โมเดลปรับตัวตามข้อมูลและสภาพอากาศที่เปลี่ยนแปลง
